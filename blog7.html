Blog post #7: Redux

Like you mentioned in your lecture 19 post, I decided to investigate Redux.

First, I went and searched for why I should use Redux in the first place. I found this article which covered what you mentioned in your lecture about a store keeping the application's state along with a how-to-use recap. One thing I liked about the article is that it also talked about when not to use Redux.

Next, like your mentioned, I looked for sources that use the newer hook system(useSelector and useDispatch). However, it was at this point that I realized I don't really know what a hook is. I remembered watching your videos implementing hooks, but I never really investigated further.

I went to the reactjs page and read their Hooks at a Glance page which caught me up along with their api reference page. Their overview summarized it nicely as "Hooks are functions that let you 'hook into' React state and lifecycle features from function components. Hooks don’t work inside classes — they let you use React without classes".

At this point I felt confident to tackle Redux's hooks api page.
After reading up on it, I found this article which did a nice side-by-side comparison of using Redux Hooks versus the previous connect way.

Below, you will find the code comparison provided by the previously mentioned article. 

                
import React from "react";
import { connect } from "react-redux";
import { addCount } from "./store/counter/actions";

export const Count = ({ count, addCount }) => {
  return (
    <main>
    <div>Count: {count}</div>
    <button onClick={addCount}>Add to count</button>
    </main>
  );
};

const mapStateToProps = state => ({
  count: state.counter.count
});

const mapDispatchToProps = { addCount };

export default connect(mapStateToProps, mapDispatchToProps)(Count);

Versus the React Redux Hook 

                
import React from "react";
import { useDispatch, useSelector } from "react-redux";
import { addCount } from "./store/counter/actions";

export const Count = () => {
  const count = useSelector(state => state.counter.count);
  const dispatch = useDispatch();

  return (
    <main>
    <div>Count: {count}</div>
    <button onClick={() => dispatch(addCount())}>Add to count</button>
    </main>
  );
};

Visually, React Redux Hooks are a lot cleaner and more concise. To me, this means that it is less error-prone which is great for me since this is my first time working with React in general.

Looking all of this up made me realize that web development is a lot different from the coding I've done at JMU. Most classes tended to avoid using any type of 'non-official' library or framework. Having the ability to quickly learn these is probably more relevant than what I've come to expect from classes which I am appreciative of.
