Blog post #8: API calls/Grabbing information

Today's blog post revolves around doing research for Project 2.

To breifly summarize, my project is basically an all-in-one news feed page. One thing I wanted to include in this was an option to allow youtube videos to appear on this page.

First, I went to here to look at what I was dealing with. I followed the instructions on the page to obtain an API key, and enabled the YouTube Data API v3.

Next, I searched for how to grab the information I needed.

After a few minutes of looking, I found this helpful stackoverflow answer. To summarize, it broke down the query options: part, chart, regionCode, key, and maxResults.

Now that I had this understanding, I was able to get the results I needed.
For now, I wanted to be able to get the 'trending' page for the USA. Specifically, I needed to get the video id to embed it into the website.

part = contentDetails
chart = mostPopular
regionCode = US
maxResults = 5

One thing I need to fully figure out is how to protect my API keys. Briefly searching brought me here, but I have yet to fully comprehend it. This might be a good thing to research in my next blog post as I will need to call multiple APIs in my project.

Next, for my project, I also wanted to be able to grab the top x amount of posts from a reddit subreddit.
This process was far more simple since you don't need an API key to grab data from reddit.

https://www.reddit.com/r/food/top/.json?count=20 is an example of how you can easily get a json file. The previous link gets the top 20 results from the /r/food subreddit.

Most information I gathered either came from their api page or from the redditdev subreddit.

After doing this research for my project, I realized that it's really handy that all of this information is freely available. It would have been much harder/annoying to make some sort of script that scraped from the normal webpages.
