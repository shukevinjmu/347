Blog post #2: JAvaScRiPT basICs

In this blog post, I wanted to go over some JavaScript basics. To better retain the information, I thought it would be better to do so with some type of creative outlet.

My first idea was to create a randomized case generator for text. For example:
"Hello, my name is Kevin", would turn into something like: "HElLo, MY nAMe iS keviN".

To do so, I went to the W3 schools javascript page and looked for a couple of things. I imagined I would need some sort of randomizer, and a string split/join or a loop.

Since I either had to make the letters upper or lower case, I created a binary randomizer which I learned through W3 schools. 

                
function binaryRandom(){
    return Math.round(Math.random());
}

Next I needed to randomize the text to either be upper or lower case. To do so I used the String.toLowerCase() and String.toUpperCase() functions.
After this, all it took was a simple loop to iterate a string with the above functions.
In the end I came up with this: 

                
function randomCaseText(){
    var input = document.getElementById("input").value;
    var output = "";
    for(const c of input){
        output += binaryRandom() ? c.toUpperCase() : c.toLowerCase();
    }
    document.getElementById("output").innerHTML = output;
}

In the above code, I have two variables: one is for input of the text, and one is for the output of the text. Next, I have a simple loop to go through the String. Inside the loop, I set the output to either upper or lower case using the binaryRandom() function I created above. Lastly, I set the textarea of element id "output" to the modifed text output.



While applying the code, I was excited to see that the input value was iterable. I dislike having to split/join strings and then iterate, which is common in certain other languages. Overall, this little exercise showed me how similar the principles of javascript are to other languages. The only learning curve I've experienced seems to be HTML DOM references.

Since doing the above was simple, I decided to try implementing "zalgo" text. Zalgo text is the process of combining regular letters with excess diacritical marks to create an amalgamated version. 

                
function zalgoCaseText() {
    var input = document.getElementById("corruptInput").value;
    var output = "";
    var len;
    for (const c of input) {
        len = random(20, 1);
        output += c;
        for (let i = 0; i < len; i++) {
            output += Z.chars[random(3, 1)][random(16, 1)];
        }
    }
    document.getElementById("corruptOutput").innerHTML = output;
}

Not included in the above code, I added a new random function that specifies a range, and the Z char array which contains the unicode to create zalgo text.
The code is mostly similar to randomCaseText(). The only difference is that I included a loop that randomly adds between 0 and 19 zalgo characters to each letter. The letter is then randomized as such: Z.chars["value between 0-2"]["value between 0-15"]
The Z char array contains 3 arrays (one each position ex: zalgo top, zalgo mid, zalgo bottom), which each have 15 zalgo characters.



Applying these text effects was a fun way to review the basics. I remember going on google a few years ago to find these annoying text effects for use in multiplayer games. I now realize that those websites were actually pretty simple to make.
